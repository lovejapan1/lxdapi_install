# Sakura LXDAPI Install

Sakura-branded LXDAPI installer for Debian/Ubuntu.

The panel display text uses `服务器`, but WHMCS compatibility is kept unchanged. The API route is still `/api/system/containers`, so existing WHMCS modules can continue to connect normally.

## One-command install

Run this on a fresh Linux server as `root`:

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/lovejapan1/lxdapi_install/main/install_all.sh)
```

This runs the full flow:

1. Install and configure LXD
2. Install Sakura panel
3. Import LXD images

## Separate commands

Install and configure LXD:

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/lovejapan1/lxdapi_install/main/lxd_install.sh)
```

Install Sakura panel:

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/lovejapan1/lxdapi_install/main/lxdapi_install.sh)
```

Import images:

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/lovejapan1/lxdapi_install/main/image_import.sh)
```

Update panel binary:

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/lovejapan1/lxdapi_install/main/lxdapi_update.sh)
```

## WHMCS settings

Use the values printed after installation:

```text
Host/IP: your-server-ip
Port: 8443
Access Hash: installer API key
```

API test:

```bash
curl -k -H "X-API-Hash: YOUR_API_KEY" https://127.0.0.1:8443/api/system/containers
```

Expected working response:

```json
{"code":200,"msg":"success","data":[]}
```

If WHMCS is on another server, test with the public IP:

```bash
curl -k -H "X-API-Hash: YOUR_API_KEY" https://YOUR_SERVER_IP:8443/api/system/containers
```

## Notes

- ACME is disabled by default because IP-only installs often fail certificate verification.
- Self-signed HTTPS is normal for WHMCS if the module uses `-k`/SSL verify off behavior.
- The installer creates `/usr/local/bin/lxc` pointing to `/snap/bin/lxc`.
- The systemd service includes `/snap/bin` in `PATH` so the API can call `lxc`.
- Do not rename API paths from `containers`; only the visible Chinese UI label was changed to `服务器`.

## Kept files

- `install_all.sh`: one-command installer for LXD, Sakura panel, and image import.
- `lxd_install.sh`: LXD installer and network/storage configurator.
- `lxdapi_install.sh`: Sakura panel installer.
- `lxdapi_update.sh`: updater that keeps existing config and checks LXD before restart.
- `image_import.sh`: LXD image import helper.
- `debian_zfs.sh`: Debian OpenZFS build helper used by `lxd_install.sh`.
- `lxdapi-linux-amd64.tar.gz`: rebuilt amd64 package generated by GitHub Actions.
- `lxdapi-linux-arm64.tar.gz`: rebuilt arm64 package generated by GitHub Actions.
- `.github/workflows/rebuild-packages.yml`: package rebuild workflow.

Obsolete one-off fix scripts were removed because their fixes are now covered by the installers and updater.
